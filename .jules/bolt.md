## 2024-04-22 - Optimize PATH traversal avoiding `stat` system calls
**Learning:** In Rust's standard library, `fs::read_dir` yields directory entries. Calling `entry.path().is_file()` inside the loop reconstructs a full path and triggers a `stat` system call on the filesystem for each entry. In operations traversing the entire `$PATH` with thousands of executables, these redundant `stat` calls become a massive performance bottleneck (~1.4s vs ~30ms).
**Action:** Always prefer `entry.file_type()` which accesses the file type information cached in the directory entry itself (on most Unix systems), avoiding extra disk I/O. For symlinks (`file_type.is_symlink()`), fallback to `entry.path().is_file()` if you need to know what they resolve to. Use `entry.file_name()` instead of `entry.path().file_name()` to avoid allocating temporary `PathBuf` objects.
