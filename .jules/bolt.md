## 2024-06-25 - Avoid redundant `stat` syscalls in directory traversal
**Learning:** In Rust's `fs::read_dir`, calling `entry.path().is_file()` always constructs a new `PathBuf` and performs a `stat` syscall to check the file type. For optimal directory traversal on OSes that provide file type information in directory entries, `entry.file_type()` gets the metadata directly from the directory entry, avoiding the `stat` overhead (with a fallback for symlinks). Additionally, `entry.file_name()` avoids the `PathBuf` allocation entirely compared to `entry.path().file_name()`.
**Action:** When iterating over directory entries (especially in large directories like the ones in `$PATH`), always use `entry.file_type()` and `entry.file_name()` instead of methods that require `entry.path()` to minimize memory allocations and expensive I/O operations.
