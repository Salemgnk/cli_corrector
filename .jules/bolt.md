## 2024-05-24 - [Rust FileSystem Performance: Avoiding `stat` Calls]
**Learning:** Using `entry.path().is_file()` during `fs::read_dir` traversal incurs a significant performance overhead because it triggers an extra `stat` system call for every entry. `entry.file_type()` gets the file type directly from the directory entry (e.g., via `getdents64` on Linux) without needing the extra syscall. However, it does not follow symlinks, so fallback is needed if `is_symlink()` is true, especially for `$PATH` directories that frequently contain symlinks. Also, getting the file name directly via `entry.file_name()` instead of `entry.path().file_name()` prevents allocating a new `PathBuf`.
**Action:** When traversing directories in Rust, always prefer `entry.file_type()` over `entry.path().is_file()`, falling back to `entry.path().is_file()` only if `file_type.is_symlink()` is true. Furthermore, use `entry.file_name()` instead of `entry.path().file_name()` to avoid unnecessary heap allocations.
