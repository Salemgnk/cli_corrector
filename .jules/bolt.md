## 2024-04-16 - [Performance Improvement: Avoid redundant stat calls when reading directories]
**Learning:** In Rust's `std::fs::read_dir`, checking if a path is a file using `entry.path().is_file()` issues a redundant `stat` system call because it resolves the whole path to check the metadata. Using `entry.file_type()` directly reads from the directory entry if supported by the OS, which avoids an extra system call. Additionally, `entry.file_type()` requires a fallback to `entry.path().is_file()` if the file is a symlink, since `is_symlink()` is true and does not imply it points to a file without resolution. Also, using `entry.file_name()` directly instead of `entry.path().file_name()` avoids an unnecessary `PathBuf` allocation.
**Action:** Always prefer `entry.file_type()` for type checking during directory traversal and `entry.file_name()` for names, keeping symlink edge cases in mind.
