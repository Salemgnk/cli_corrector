## 2024-06-25 - Avoid redundant stat calls and allocations during directory traversal
**Learning:** In Rust's `fs::read_dir`, calling `entry.path().is_file()` triggers an underlying `stat` system call. Traversing directories like those in `$PATH` this way is heavily unoptimized. Instead, `entry.file_type()` fetches file metadata already available from the `readdir` call on most OSes. A fallback to `entry.path().is_file()` is only required for symlinks. Additionally, using `entry.path().file_name()` allocates a new `PathBuf` for every file, whereas `entry.file_name()` returns an `OsString` directly from the entry, saving memory allocations.
**Action:** Always prefer `entry.file_type()` over `entry.path().is_file()` when iterating over `fs::read_dir`, with explicit handling for symlinks. Use `entry.file_name()` instead of constructing the full path if only the file name is needed.
