## 2025-02-28 - Avoid `PathBuf` allocation and redundant stat in `std::fs::read_dir` loops
**Learning:** In Rust, `std::fs::read_dir` yields `DirEntry` objects which often cache the file type (especially on Unix and modern Windows). Calling `entry.path().is_file()` allocates a new `PathBuf` and makes a redundant `stat` system call to check the file type. Calling `entry.file_type()` gets the file type directly from the directory entry (when available) without allocating memory or calling `stat`. However, `file_type()` **does not follow symlinks**. If an entry is a symlink, `file_type.is_symlink()` is true, and `file_type.is_file()` is false. When traversing paths like `$PATH` where symlinked executables are common, failing to follow symlinks leads to ignoring critical files.
**Action:** When traversing directories using `std::fs::read_dir` to find files, prefer using `entry.file_type()` to check for file type to avoid unnecessary system calls. However, always check `file_type.is_symlink()` and fall back to `entry.path().is_file()` if it is a symlink, ensuring that symlinked files are properly identified without sacrificing the performance gain for non-symlink entries. Also, use `entry.file_name()` to get the name, avoiding `entry.path()` unless the full path is explicitly needed.
