## 2024-05-18 - Optimize executable file discovery in `load_available_commands`
**Learning:** `entry.path().is_file()` on `fs::DirEntry` forces a `stat` syscall for every file, which is highly inefficient when iterating over thousands of files in `$PATH` directories. `entry.file_type()` often retrieves the file type directly from the directory entry (where supported by OS) without requiring a `stat` call, except for symlinks where `is_file()` on the path is still needed. Similarly, using `entry.path().file_name()` creates unnecessary `PathBuf` allocations, whereas `entry.file_name()` retrieves it efficiently.
**Action:** Always prefer `entry.file_type()` over `entry.path().is_file()` when iterating directories to avoid expensive `stat` operations, and use `entry.file_name()` directly to avoid `PathBuf` allocations.
