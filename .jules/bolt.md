## 2024-05-18 - Rust Directory Traversal Performance
**Learning:** `std::fs::read_dir` traversal using `entry.path().is_file()` incurs a severe performance penalty because `entry.path()` creates a full `PathBuf` allocation, and `is_file()` triggers a `stat` system call. The system's `$PATH` can contain thousands of files, amplifying this bottleneck significantly.
**Action:** Always prefer `entry.file_type()` which utilizes the metadata already retrieved by `read_dir` on most platforms, avoiding the `stat` call. However, since `file_type()` doesn't follow symlinks, it must be combined with a fallback to `entry.path().is_file()` specifically when `file_type.is_symlink()` is true. Also, prefer `entry.file_name()` over `entry.path().file_name()` to avoid the `PathBuf` allocation entirely.
