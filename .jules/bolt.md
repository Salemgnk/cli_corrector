## 2024-05-15 - Optimize directory traversal in $PATH
**Learning:** In Rust's `std::fs::read_dir`, calling `entry.path().is_file()` inside a loop allocates a new `PathBuf` and triggers a blocking `stat` system call on every file. This is extremely slow when scanning thousands of binaries in the $PATH environment variable. By checking `entry.file_type()` instead, which caches file metadata on many operating systems, we can skip the `stat` call entirely for non-symlinks. Similarly, `entry.file_name()` returns an `OsString` without allocating a full path.
**Action:** Always prefer `DirEntry::file_type()` and `DirEntry::file_name()` over `DirEntry::path()` when inspecting directory contents to avoid unnecessary allocations and blocking disk I/O. Only fallback to `path().metadata()` or `path().is_file()` if the file type indicates a symlink.
